# LAO 핸드오프 메커니즘

Date: 2026-05-23

이 문서는 LAO에서 생성된 기획/설계 명세서를 개발 AI 에이전트(Claude Code, Codex 등)에 전달하는 방식(Handoff)을 설명한다.

---

## 1. 개요

LAO 0.9 버전에서는 기존 Swift 기반의 MCP 서버(`LAOMCPServer`) 및 macOS 전용 `.command` 파일 실행 스크립트 기반의 복잡한 연동 구조가 제거되었습니다. 대신 **호환성이 높고 직관적인 통합 마크다운 컴파일 모델**로 대체되었습니다.

핸드오프 절차는 다음과 같이 단순하게 작동합니다:
1. **기획 명세서 컴파일** — 사용자가 작성한 모듈식 명세서(`.lao/specs/` 폴더 내 마크다운 파일들)와 결정된 아키텍처 의사결정 내역은 Express 백엔드 서버에 의해 하나의 마크다운 기획 문서(`spec_compiled.md`)로 자동 수합 및 컴파일되어 `.lao/` 폴더에 기록됩니다.
2. **리뷰 및 추출** — Web UI의 우측 **Live Compiled Spec** 탭에서 다음 기능을 지원합니다:
   * **Monospace 로그 뷰어**: 실시간 컴파일된 마크다운을 텍스트 뷰로 확인.
   * **Copy**: 전체 명세서 내용을 클립보드에 원클릭 복사.
   * **Download**: 로컬 환경에 `spec_compiled.md` 파일로 다운로드.
3. **AI 접수** — 복사된 텍스트나 다운로드한 명세서 파일을 에이전트(Claude Code, Codex 등)에게 제공하여 즉각적인 구현을 지시합니다.

---

## 2. 기획 명세서 컴파일 사양

* **산출 경로**: `{project_root}/.lao/spec_compiled.md`
* **호출 API**: `POST /api/specs/compile` (Web UI의 기획서 실시간 업데이트 또는 편집 완료 시 백엔드에서 자동 호출 및 동기화)

컴파일 엔진은 `core_spec.md` 파일과 `.lao/specs/features/` 아래의 활성화된 개별 기능 명세 파일들을 취합하여 다음과 같은 마크다운 형식으로 구성합니다:
* 프로젝트의 핵심 목표, 기술 스택 및 Golden Rules(기술 가드레일 제약 조건)
* 세부 시스템 명세서 (사용자 흐름, 컴포넌트 세부 기능 정의 등)
* 의사결정 카드를 해결하여 기록된 최종 타임라인 및 결정 근거 로그

---

## 3. CLI AI 에이전트 연동 방법 (Handoff Flow)

컴파일 및 추출된 설계 명세서를 로컬 터미널의 AI 도구들에게 전달하여 개발을 지시하는 방법은 다음과 같습니다:

### 방법 A: 로컬 파일 직접 지정 (권장)
터미널에서 AI 도구를 기동할 때, 생성된 명세서 경로를 프롬프트 매개변수로 명시하여 프로젝트 구현을 지시합니다.

* **Claude Code**:
  ```bash
  claude -p "Read .lao/spec_compiled.md and implement the requirements."
  ```
* **Codex CLI**:
  ```bash
  codex exec "Implement the features specified in .lao/spec_compiled.md"
  ```

### 방법 B: 마크다운 본문 직접 복사
1. LAO Web UI의 우측 **Spec** 탭으로 이동합니다.
2. **Copy** 버튼을 클릭하여 전체 명세서를 클립보드에 복사합니다.
3. 실행 중인 AI 에이전트와의 채팅창에 붙여넣어(Paste) 컨텍스트로 주입하고 구현을 요청합니다.

---

## 4. 연동 모델 간소화 배경

기존 Swift MCP 서버 아키텍처에서 텍스트/파일 기반 마크다운 핸드오프로 간소화한 세 가지 요인은 다음과 같습니다:
1. **설치 환경 의존성 제로**: Xcode, Swift 패키지 컴파일, 혹은 `.mcp.json` 등록 등의 호스트 환경 설정을 요구하지 않으므로 Windows, Linux, macOS 환경 구분 없이 완벽하게 동작합니다.
2. **최신 AI 에이전트 호환성**: 대부분의 CLI 개발 AI들은 정적 문서나 복사된 마크다운을 직접 주입받아 이해하는 능력이 대단히 뛰어나므로, 추가적인 MCP stdio 통신 레이어가 없어도 고품질 코드를 완벽하게 작성해 냅니다.
3. **샌드박스 및 권한 이슈 방지**: 단일 마크다운 파일로 입출력을 제어하므로 로컬 디렉터리 권한 제한이나 OS 수준의 앱 보안 샌드박스로 인한 실행 실패를 원천적으로 방지합니다.
