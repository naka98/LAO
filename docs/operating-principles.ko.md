# LAO 운영 원칙 v0.9

Date: 2026-05-27

## 1. 목적

LAO(Leeway AI Office)는 **원시 아이디어를 구조화되고 AI 실행 친화적인 설계 명세서로 변환하는 워크스페이스**입니다.

로컬 CLI 기반의 특화된 AI 에이전트들을 오케스트레이션하여 모듈식 명세서를 생성하고, 주요 아키텍처 선택 항목을 인터랙티브 카드 형태로 제시하며, 이를 하나의 통합 기획 명세서(`spec_compiled.md`)로 빌드하여 개발 단계로 넘겨줍니다.

```
아이디어 구상 → 명세서 자동 Sprout → 의사결정 카드 해결 → 스펙 잠금 (Lock) → 개발 및 검증 (DevLoop)
```

역할 분리:
* **LAO** = 기획 설계 전환기 — *무엇을 만들지* 정의하고 기술 스택 가드레일을 통제합니다.
* **개발 AI** (예: Claude Code, Codex) = 구현기 — 잠금 처리된 `spec_compiled.md` 파일을 읽고 실제 코드를 작성합니다.

---

## 2. 핵심 철학

### 2.1 문서 기반 워크스페이스
개발자의 인지적 피로도를 최소화하기 위해, 복잡한 visual 마인드맵 조작을 지양합니다. 워크스페이스는 분할 화면 구조를 취하며, 실시간 명세서 문서 뷰어와 고수준의 아키텍처 의사결정 카드를 중심으로 단순하게 진행됩니다.

### 2.2 Golden Rules (기술 스택 가드레일)
AI가 부적절한 설계를 제안하는 것을 막기 위해 강력한 기술적 제약 사항(`lao.config.json` 내 `goldenRules`)을 주입합니다. 모든 에이전트 프롬프트에 이 가드레일이 들어가므로 기획 명세가 일관되게 규격화됩니다 (예: SQLite 사용, Docker 비의존).

### 2.3 Phase Lock 프로토콜
개발 진행 중 기획 명세와 실제 코드 간 불일치(Drift)가 일어나는 것을 방지합니다:
* **Planning Phase (기획 단계)**: 명세서의 자동 Sprout 및 직접 인라인 편집, 의사결정 카드의 선택이 모두 열려 있습니다.
* **Development Phase (개발 단계)**: 기획 명세서가 읽기 전용 상태로 잠깁니다. 빌드, 실행, 검증 명령어를 Web UI 내의 DevLoop 가상 콘솔에서 직접 수행합니다.

## 3. 디렉터리 및 저장 구조

프로젝트 루트의 `.lao/` 폴더 하위에 파일 단위로 정보가 관리됩니다:

```text
.lao/
├── lao.config.json           # 기술 제약 가드레일, 단계(Phase), 프로바이더 설정
├── criteria.md               # 채택한 의사결정의 이력 및 근거 기록 파일
├── spec_compiled.md          # 코어 및 기능별 기획 명세서가 하나로 컴파일된 파일
├── messages.json             # 에이전트 팀과 나눈 대화 로그
├── specs/
│   ├── core_spec.md          # 코어 아키텍처 명세서
│   └── features/
│       └── [feature_id].md   # 개별 기능 명세서 (YAML frontmatter + 마크다운 바디)
└── decisions/
    └── [card_id].json        # Optionizer가 제안한 대안 선택 의사결정 카드 파일
```

## 4. 다중 에이전트 역할 정의

LAO는 고유의 페르소나를 가진 에이전트들을 통해 기획을 고도화합니다:

1. **Director**: 오케스트레이터. 사용자의 메시지 라우팅, 스텝 태스크의 분할 및 전체 기획/개발 단계 상태를 총괄합니다.
2. **Specifier**: 명세서 작성 담당. `.lao/specs/features/` 및 `core_spec.md` 명세 초안을 자동으로 구조화하여 생성(Sprout)합니다.
3. **Optionizer**: 아키텍처 대안 제시 담당. Sprout된 문서들을 분석하여 사용자가 판단해야 할 의사결정 카드(`decisions/*.json`)들을 생성합니다.
4. **Gap Detector**: 누락/오류 검출 담당. 사양서 간 불일치나 논리적 공백을 추적하여 UI 상에 경고 문구를 띄워줍니다.

---

## 5. 개발 루프 (DevLoop) 연동

설계가 끝나 개발 단계로 잠금 전환되면, 개발자는 Web UI의 **DevLoop Console** 탭을 통해 빌드, 서버 실행, E2E 검증 테스트 명령어를 직접 실행할 수 있습니다. 셸 출력 스트림(stdout/stderr)이 가상 콘솔 화면에 실시간으로 출력되므로, 코드가 잠금 처리된 명세 기준에 부합하게 작성되는지 직관적으로 검증할 수 있습니다.
